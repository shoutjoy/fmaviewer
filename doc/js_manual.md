# FMA Viewer JavaScript Manual

이 문서는 FMA Viewer Ultra의 모듈화된 JavaScript 구조와 주요 기능에 대한 기술 매뉴얼입니다. 모든 로직은 성능 최적화와 유지보수 편의를 위해 `js/` 폴더 내의 각 역할별 파일로 분리되어 있습니다.

---

## 📂 파일 구조 및 역할 (File Architecture)

### 1. `js/globals.js` - 전역 상태 및 DOM 캐시
애플리케이션 전체에서 공유되는 데이터와 DOM 요소 참조를 관리합니다.
- **이미지 데이터**: `images` 배열 (이미지 소스, 경로, 그룹, 즐겨찾기 상태 등 포함)
- **현재 상태**: `currentIndex` (현재 선택된 이미지 인덱스), `zoom` (확대 배율), `viewMode` (Single/Two), `orientation` (Horz/Vert) 등
- **DOM 캐시 (`dom` 객체)**: 빈번한 `document.getElementById` 호출을 방지하여 성능을 극대화합니다.

### 2. `js/database.js` - 데이터 보관 (IndexedDB)
브라우저 종료 후에도 작업을 이어갈 수 있도록 데이터를 관리합니다.
- **자동 저장**: 이미지 추가, 삭제, 즐겨찾기 변경 시 즉시 IndexedDB에 반영됩니다.
- **세션 복구**: `initDB()`를 통해 마지막에 열었던 프로젝트 데이터를 자동으로 불러옵니다.

### 3. `js/gallery.js` - 좌측 리스트 및 갤러리 로직
갤러리 화면 구성과 썸네일 상호작용을 담당합니다.
- **성능 최적화**: `DocumentFragment`를 활용하여 수백 개의 이미지를 한 번에 렌더링하며, `loading="lazy"`를 적용하여 하드웨어 부담을 줄였습니다.
- **정렬 기능**: 그룹별(Default), 생성날짜순, 파일 크기순 정렬을 제공합니다.
- **즐겨찾기 화면**: 별도의 사이드바 영역에 즐겨찾기된 이미지만 모아서 보여줍니다.
- **복구 시스템**: `deletedImages` 스택을 통해 실수로 삭제한 이미지를 역순으로 복구하는 기능을 포함합니다.

### 4. `js/preview.js` - 미리보기 및 메타데이터 렌더링
우측 큰 화면 표시와 관련 상세 로직입니다.
- **하단 메타데이터 바**: 현재 보고 있는 이미지의 경로를 표시하며, **각 이미지별 전용 `★ Favorite` 및 `Download` 버튼**을 동적으로 생성합니다.
- **듀얼 뷰(Two) 대응**: 2페이지 모드 시 하단 바가 좌우로 나뉘어 각각의 이미지 정보를 독립적으로 표시합니다.
- **방향 전환**: 가로 슬라이딩(Horz)과 세로 스크롤(Vert) 모드 간의 렌더링 전환을 처리합니다.

### 5. `js/fileHandlers.js` - 파일 처리 및 내보내기
외부와의 데이터 입출력을 담당합니다.
- **FMA 로드**: JSON 기반의 `.fma` 파일을 해석하여 내부 데이터 구조로 변환합니다.
- **이미지 추가**: 로컬 파일을 드래그하거나 선택하여 현재 프로젝트에 병렬로 추가합니다.
- **내보내기**: 현재 작업 중인 상태를 `.fma` 파일로 다시 저장하거나, 모든 이미지를 한꺼번에 `.zip` 파일로 묶어서 다운로드할 수 있습니다.

### 6. `js/interaction.js` - 사용자 상호작용 (Zoom & Pan)
마우스와 키보드를 통한 조작 로직입니다.
- **확대/축소 (Zoom)**: 상단 버튼 및 `Ctrl + 마우스 휠`을 통해 실시간 확대/축소가 가능합니다.
- **드래그 이동 (Pan)**: 확대된 상태에서 마우스를 클릭한 채 드래그하여 이미지의 특정 부분을 탐색합니다.
- **단축키**: 좌우 화살표 키를 이용한 빠른 이미지 탐색을 지원합니다.
- **패널 리사이즈**: 중앙 바를 조절하여 좌측 갤러리와 우측 미리보기 영역의 너비를 조정하고 이를 기억합니다.

### 7. `js/app.js` - 메인 제어 (Orchestrator)
모든 모듈을 초기화하고 이벤트 리스너를 결합하는 중심 파일입니다.
- **초기화 순서**: 그리드 설정 -> 데이터베이스 연결 -> 이벤트 바인딩 -> 모드 업데이트 순으로 실행됩니다.
- **이벤트 중앙 관리**: 상단 툴바의 모든 버튼 클릭 이벤트를 각 기능 파일의 함수와 연결합니다.

---

## 🚀 주요 성능 개선 포인트
1. **DOM Access 병목 해소**: 모든 주요 요소를 `dom` 객체에 사전 캐싱하여 접근 속도를 높였습니다.
2. **배치 렌더링**: 화면 업데이트 빈도를 최소화하기 위해 메모리 내에서 DOM 구조를 완성한 뒤 배포합니다.
3. **비동기 처리**: 대용량 이미지 로드 시 UI가 멈추지 않도록 비동기 전송 방식을 사용합니다.
